# 奇点纪元 (Singularity Era)

AI 公司模拟经营游戏。玩家从零创建一家 AI 公司，经历研发、训练、发布、市场竞争的完整流程，最终打造通用人工智能。

## 技术栈

- **前端框架**: React 18 + TypeScript (strict mode)
- **构建工具**: Vite 6
- **状态管理**: Immer (不可变状态更新)
- **样式**: CSS Modules
- **架构**: ECS 启发式设计 (Entity-Component-System)

## 项目结构

```
src/
├── core/                          # 游戏核心逻辑（纯 TypeScript，无 React/DOM 依赖）
│   ├── commands/                  # 命令模式：所有玩家操作封装为 Command
│   ├── config/                    # 配置文件：资源、硬件、地区、科技树等
│   ├── entities/                  # 实体定义：员工、模型、数据集、基础设施等
│   ├── interfaces/                # 接口：Command、System
│   ├── resources/                 # 资源注册系统
│   ├── systems/                   # 13 个游戏系统（每日按顺序执行）
│   ├── utils/                     # 工具函数：效率计算、市场计算、跨系统联动
│   ├── EventBus.ts                # 事件总线
│   ├── Game.ts                    # 游戏入口
│   ├── GameLoop.ts                # 游戏主循环（requestAnimationFrame）
│   └── GameState.ts               # 游戏状态定义
├── ui/                            # 用户界面层
│   ├── components/                # React 组件（11 个面板）
│   ├── context/                   # React Context (GameProvider)
│   ├── hooks/                     # 自定义 Hooks (useGame, useGameState)
│   └── styles/                    # CSS Modules 样式
└── main.tsx                       # 应用入口
```

## 核心架构

### 游戏循环

```
GameLoop (requestAnimationFrame)
  → 累积 deltaMs → 达到 1 天时触发 advanceDay()
  → 按顺序执行 13 个 System.update()
```

### 系统执行顺序

| 序号 | 系统 | 职责 |
|------|------|------|
| 1 | ComputeHardwareSystem | 硬件交付、磨损、故障 |
| 2 | PowerSystem | 电力计算、超容自动从电网购电 |
| 3 | TechResearchSystem | 科技树研究进度推进 |
| 4 | ResearchSystem | 模型研发项目进度、实验 |
| 5 | CollectionSystem | 数据收集项目进度 |
| 6 | InfrastructureFailureSystem | 基础设施随机故障 |
| 7 | TrainingSystem | 模型训练进度、事件 |
| 8 | InfraMaintenanceSystem | 基础设施维护成本 |
| 9 | OperationsSystem | 运营收入、用户增长 |
| 10 | CompetitorSystem | 竞争对手 AI 决策（每 7 天） |
| 11 | RiskSystem | 风险事件、法律/信任负债 |
| 12 | RegionSystem | 地区税收、市场扩展 |
| 13 | StaffSystem | 薪资发放、绩效评估、忠诚度 |

### 命令模式

所有玩家操作通过 `game.executeCommand(new XxxCommand(...))` 执行，确保状态变更可追踪。

### 跨系统联动

通过 `crossSystemUtils.ts` 集中管理员工属性对其他系统的影响：
- 员工效率 → 训练加速、研发加速、故障降低、收入倍率、风险降低
- 部门加成 → 部门负责人的领导力影响部门效率
- 公司协同 → 所有负责人领导力均值影响全公司

## 已实现系统

### 资源与基础设施
- 7 种算力卡型号 (L40S → GB300)，含 CardInstance 实例管理
- 数据中心 → 集群 → 节点 → 计算卡 四级基础设施层级
- 电力：自建电站 + 电网自动购电（按地区能源成本定价）
- 云算力租赁：6 家匿名化云服务商 (Nimbus/Stratus/Cirrus/Nova/Aurora/Tenji)

### 员工系统
- 5 种核心员工角色：研究员、数据工程师、系统工程师、产品经理、法务/公关
- 5 种普通员工（资源计数管理）
- 等级体系 L1-L10，属性系统（智力/创造力/领导力/耐力/魅力）
- 招聘渠道（校招/招聘网站/猎头/内部推荐）
- 培训系统（技能/高级/海外）
- 部门体系（5 个部门，负责人任命）
- 绩效管理（30 天评估周期，S/A/B/C 等级）
- 激励机制（奖金/股权/团建）

### 模型训练
- Chinchilla 缩放定律计算训练算力需求
- 长序列修正因子
- 模型并行（自动检测显存不足）
- 训练事件（spike/loss spike 等）
- 模型发布与能力评估

### 数据系统
- 多数据集管理（创建/删除/切换）
- 数据自收集（数据工程师驱动）
- 数据购买（冷却期机制）
- 数据质量计算

### 地区与市场
- 33 个全球地区，含人口/GDP/监管/人才/能源等属性
- 地区税收、数据本地化要求
- 市场渗透、竞争对手模拟
- 语言适配（中文名/英文名）

### 运营与竞争
- 模型 API 运营（Token 收入）
- 融资系统（种子轮/A 轮/B 轮/IPO）
- 竞争对手 AI（融资/训练/发布/合并）
- 激进操作（收购/挖角/渗透/黑客）
- 外部企业渗透

### 风险系统
- 法律负债、信任负债
- 员工士气、声望
- 对齐等级
- 风险事件触发

### 科技树
- 两阶段科技树（P0 基础 + P1 进阶）
- 研究进度、解锁效果

## 开局流程

1. **选择总部地区** — 33 个地区，推荐高经济水平地区
2. **选择开局策略** — 3 个预设（自力更生/均衡起步/技术驱动）+ 自定义配置
3. **自定义配置** — 独立选择资金($1M-$20M)、算力卡(A100/H100/L40S)、员工(5 种角色 + 等级)

## 设计原则

1. **核心逻辑与 UI 分离** — `src/core/` 纯 TypeScript，可独立测试
2. **配置驱动** — 新资源/硬件/地区通过配置文件添加，无需修改核心代码
3. **命令模式** — 所有状态变更通过 Command，便于追踪和回滚
4. **系统解耦** — 13 个系统通过事件总线通信，低耦合高内聚
5. **不可变状态** — Immer 管理状态更新，防止意外修改
